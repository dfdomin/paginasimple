Rúbrica de Evaluación HTML & CSS

Criterio 1: Estructura y Etiquetas HTML (CE1, RA4)

Ponderación máxima: 1.25 puntos

Aspectos a evaluar

  • Uso de etiquetas semánticas (header, main, section, footer, etc.)
  • Empleo de texto, hiperenlaces, tablas, formularios, imágenes.
  • Correcto anidamiento (buena práctica).
  • Se evidencia estructura inspirada en Cat Photos (uso de <img>, <section>, etc.).

Tabla de nivel de logro

Nivel Descripción Puntaje
Excelente (1.25) Emplea de manera óptima la estructura semántica (header, main, section, footer, etc.), y todas las etiquetas básicas (enlaces, tablas, formularios, imágenes) están presentes y bien anidadas. El HTML es muy claro. 1.25
Bueno (1.0) Cubre la mayoría de etiquetas de HTML (texto, enlaces, imágenes, tablas, formularios) con pocos errores en su organización. No se detectan problemas graves de anidamiento o semántica. 1.0
Aceptable (0.75) Usa etiquetas mínimas para el funcionamiento, pero se observan omisiones notables (falta alguna sección semántica o un formulario/tipo de elemento). Presenta errores de estructura que no rompen la página, pero restan claridad. 0.75
Deficiente (0.5) HTML desordenado y escaso. Falta la mayoría de elementos solicitados (no hay tablas, imágenes o formularios) o hay múltiples errores graves de semántica/anidamiento. 0.5

Criterio 2: Estilos CSS y Maquetación (CE2, RA4)

Ponderación máxima: 1.25 puntos

Aspectos a evaluar

  • Aplicación de hojas de estilo internas o externas (color, tipografías, posicionamiento).
  • Maquetación: uso de contenedores, ancho (width), márgenes, paddings, fondo (background).
  • Inspiración en el proyecto “Menú del Restaurante” para centrar, manejar un color de fondo, etc.

Tabla de nivel de logro

Nivel Descripción Puntaje
Excelente (1.25) Uso avanzado de selectores (clases/ID) con propiedades de posicionamiento, colores y tipografías coherentes. Diseño limpio y responsive básico. Se aprecian varios aspectos aprendidos en el proyecto de menú (centrado, background, etc.). 1.25
Bueno (1.0) Aplica correctamente la mayoría de reglas CSS (tipografía, color, maquetación). Puede presentar algunos pequeños desajustes pero en general cumple con lo esperado. 1.0
Aceptable (0.75) Emplea reglas CSS básicas sin demasiada coherencia visual (colores/espacios). No se aprovechan completamente los ejemplos de menú, aunque sí hay un mínimo de estructura y estilos. 0.75
Deficiente (0.5) Apenas contiene estilos o están sumamente desordenados. No se evidencian los patrones básicos de maquetación (centrado, uso de background, tipografías). 0.5

Criterio 3: Usabilidad y Accesibilidad (RA1)

Ponderación máxima: 1.25 puntos

Aspectos a evaluar

  • Accesibilidad: uso de alt en imágenes, label en formularios, encabezados lógicos, lang en la página, enlace “Saltar al contenido”, etc.
  • Usabilidad: menú de navegación claro, colores con contraste adecuado, respuesta en dispositivos móviles, orden semántico de secciones.
  • Deben observarse 10 elementos en total (5 de accesibilidad + 5 de usabilidad), inspirados en buenas prácticas.

Tabla de nivel de logro

Nivel Descripción Puntaje
Excelente (1.25) Cumple plenamente con los 10 elementos solicitados: alt en todas las imágenes, label y semántica en formularios, un enlace para saltar al contenido, menú o índices claros, colores contrastantes. Diseño responsive que facilita la navegación. 1.25
Bueno (1.0) Aplica la mayoría de esos elementos (7-8) con algunos detalles por pulir (p.ej., un alt descriptivo faltante o contraste dudoso). No afecta gravemente la accesibilidad o usabilidad. 1.0
Aceptable (0.75) Cumple ciertos aspectos básicos (3-5) pero faltan varios puntos importantes (p.ej., no hay skip link, label asociadas, alt incompletos). La usabilidad del menú es funcional, aunque básica. 0.75
Deficiente (0.5) No aplica casi ningún principio de accesibilidad (sin alt, sin label) y la usabilidad es limitada (difícil navegación, colores muy contrastantes o muy bajos, sin responsividad). 0.5

Criterio 4: Animaciones y Requisitos del Proyecto (CE3, RA4)

Ponderación máxima: 1.25 puntos

Aspectos a evaluar

  • Animaciones básicas con transiciones o transformaciones (hover, color, escalado, etc.).
  • Requisitos finales del proyecto de hoja de vida:
    • Secciones de texto (perfil, proyectos, etc.)
    • Imágenes/capturas de proyectos
    • Tabla de tecnologías
    • Formulario de contacto
    • Ejemplo de uso de background
    • Referencia a ejemplos de Cat Photos y Menú

Tabla de nivel de logro

Nivel Descripción Puntaje
Excelente (1.25) Incluye al menos una animación o transición reconocible (hover en tarjetas, botones, etc.) y cumple todas las secciones indicadas (texto personal, proyectos con capturas, tabla de tecnologías, formulario). Uso adecuado de fondo o color de fondo, reflejando los ejemplos de clase. 1.25
Bueno (1.0) Muestra animaciones sencillas (hover o cambio de color en enlaces). Incluye la mayoría de secciones solicitadas (quizá alguna menor incompleta), pero cumple el objetivo principal. 1.0
Aceptable (0.75) Mínimo de animación (un simple hover en un botón). Faltan una o dos secciones clave (p.ej., no se incluyó la tabla o el formulario) o las capturas de proyectos son escasas. 0.75
Deficiente (0.5) No presenta animaciones/transiciones. No cumple varios de los requisitos (sin formulario, sin capturas, sin tabla de tecnologías). Presenta un contenido muy reducido. 0.5
